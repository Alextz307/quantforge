# `src/benchmarking/`

Performance-benchmark harness around Google Benchmark (`quant_bench`)
and Python hybrid timings. Captures hardware + git provenance, persists
runs as one-JSON-object-per-file JSONL, detects regressions vs.
committed baselines, and renders the same booktabs / matplotlib stack
used by the experiment reporters.

## Public surface

| Symbol | Role |
| --- | --- |
| `BenchmarkRunner.run(...)` | Drives `quant_bench --benchmark_format=json` + zero-arg Python hybrid callables; returns a `BenchmarkRun`. |
| `BenchmarkStore` | JSONL persistence under `benchmark_results/`. `save_run` writes ephemeral runs; `save_baseline(name, ...)` writes a committed anchor (refuses to clobber). |
| `BenchmarkAnalyzer.detect_regressions(current, baseline, ...)` | Two-gate rule (|pct_delta| ≥ threshold AND |z| ≥ threshold) → list of `RegressionReport`. Also `summarise(run)` and `fit_scaling_laws(run)`. |
| `BenchmarkComparator.compare(baseline, current, ...)` → `ComparisonReport` | Convenience wrapper that splits results into `regressions` / `improvements`. |
| `BenchmarkReporter.generate_full_report(...)` | Per-family scaling plots, regression bars, components plot, summary / regression / scaling LaTeX tables. |
| `BenchmarkRun`, `BenchmarkResult`, `HardwareInfo`, `BenchmarkStats`, `RegressionReport`, `ScalingAnalysis`, `ComparisonReport` | Frozen dataclasses with JSON `to_dict` / `from_dict` round-trip. |
| `SCALING_O_N`, `SCALING_O_N_LOG_N`, `SCALING_O_N_SQUARED`, `SCALING_UNCLEAR` | Scaling-law classification labels. |

## Layout

| File | Role |
| --- | --- |
| `runner.py` | Orchestrator: shells out to `cpp/build/benchmarks/quant_bench`, parses JSON, runs Python hybrid callables, captures `HardwareInfo`. |
| `store.py` | `BenchmarkStore` (`runs/` + `baselines/` directories under one root). |
| `analyzer.py` | Pure-pandas summary, regression detection, log-log scaling fit. |
| `comparator.py` | Thin presentation wrapper over `BenchmarkAnalyzer`. |
| `reporter.py` | Plots + LaTeX (Agg backend pinned via `src.visualization.plots`). |
| `types.py` | Frozen dataclasses + JSON round-trip helpers + scaling-label constants. |

## On-disk layout

```
benchmark_results/
    runs/<timestamp>_<sha7>.jsonl     # ephemeral; gitignored
    baselines/<name>.jsonl            # committed; anchors regression reports
    reports/<timestamp>/              # generated by BenchmarkReporter
        scaling_<family>.{png,svg}
        regression_vs_<baseline>.png
        components.png
        summary.tex
        regression.tex
        scaling.tex
```

## Provenance + determinism

- **`HardwareInfo` per run.** CPU brand (sysctl / /proc/cpuinfo), CPU
  count, RAM (psutil), OS, Python version, `git_sha` + `git_dirty`.
  The SHA + dirty flag anchor a measurement to a specific code state;
  baselines committed to the repo document the SHA they were captured
  on.
- **Append-only baselines.** `save_baseline` refuses to overwrite an
  existing file unless `overwrite=True` — prevents a stale rerun from
  silently invalidating a regression report.
- **No mutation after construction.** Every dataclass is `frozen=True`
  with `tuple` (not `list`) fields so the in-memory snapshot of a run
  can't drift.

## Snippet

```python
from pathlib import Path

from src.benchmarking.runner import BenchmarkRunner
from src.benchmarking.store import BenchmarkStore

runner = BenchmarkRunner(quant_bench=Path("cpp/build/benchmarks/quant_bench"))
run = runner.run(tags=("phase7", "demo"))

store = BenchmarkStore(Path("benchmark_results"))
store.save_run(run)
# Or anchor as a baseline (refuses to clobber):
# store.save_baseline(run, name="phase7_baseline")
```

## Cross-links

- Reuses `src/visualization/plots.py` (Agg backend pin + figure
  geometry) and `src/visualization/latex.py` (booktabs LaTeX builder).
- C++ benchmarks live under `cpp/benchmarks/`; the binary is built at
  `cpp/build/benchmarks/quant_bench`.
- The `make bench` target wraps the `BenchmarkRunner` invocation;
  `scripts/benchmark.py` is the user-facing CLI.
